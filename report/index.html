PPO · Procedural tracks · 2.5M steps

Two agents, one goal:
lap time vs no braking

Two identical PPO policies learned to drive a top-down racer on procedurally generated circuits. The only difference is reward shaping: one agent optimizes progress alone; the other pays a steep penalty for every deceleration. Same physics, same tracks — divergent cornering styles.

The setup

The simulator uses simple drift-friendly physics: throttle and steering, lateral grip that breaks into slides, and harsh off-track slowdown. Tracks are procedurally seeded — hairpins, chicanes, and long straights vary per seed.

Both agents train with PPO on identical observation and action spaces. The time agent maximizes forward progress (lap completion reward). The nobrakes agent gets the same signal minus λ · max(0, vt−1 − vt) — penalizing any speed drop step-to-step.

Intuition & takeaways

Reward shaping steers style, not just score

Penalizing deceleration doesn't forbid slowing down — it makes every brake expensive. The nobrakes agent finds local optima where momentum and drift substitute for classic apex braking. On open tracks it nearly matches time-only; on tight sequences it pays in lap time and off-track time.

What changed in behavior

  • Time-only: hard brakes, lower mean speed, fewer drifts, faster mean lap.
  • No-brakes: high sustained speed, more drift and off-track, fewer brake events.

Caveats

  • Single training seed per agent — results are illustrative, not a rigorous sweep.
  • 2D top-down physics simplify weight transfer and tire models.
  • Deceleration penalty is step-wise; real racing rewards line and exit speed differently.
  • Evaluation set is fixed at — procedural seeds.

CLI cheatsheet

python -m racing play --seed 7          # drive yourself
python -m racing race --vs nobrakes     # you vs an AI

python -m racing train --agent both     # retrain
python -m racing evaluate               # rebuild dataset
python -m racing report                 # rebuild charts + serve this page
